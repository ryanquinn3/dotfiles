# Planning

## Ensure plans follow style guidelines.

Read the [code style guidelines](./codestyle.md) and [testing guidelines](./testing.md) and ensure all implementation plans follow those principles.

## Verify before declaring done

Always ensure that we run lint, typecheck, and unit-test and confirm they pass before reporting a task or plan step as complete. Do not claim success without evidence.

Always make sure implementation plans include running lint, typecheck, and unit tests as part of each task. 

## Confirm code snippets are grounded

When including code examples in plans or documentation, verify they match actual codebase code. Don't fabricate snippets from memory.
